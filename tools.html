Unit testing is a type of software testing where individual units or components of a software are tested. The purpose is to validate that each unit of the software code performs as expected.



There are around 24 differnt Parts of the software process, all falling into two parts; Functional and Non-Functional testing.


  • Types of Functional Testing
  • Unit testing
  • Integration testing
  • End-to-end testing
  • Smoke testing
  • Sanity testing
  • Regression testing
  • Acceptance testing
  • White box testing
  • Black box testing
  • Interface testing

Functional testing works specifically with softwares applications to make sure it runs as intented. This is usually a mix of both manual testing and automatic testing.


  • Types of Non-Functional Testing:
  • Performance testing
  • Security testing
  • Load testing
  • Failover testing
  • Compatibility testing
  • Usability testing
  • Scalability testing
  • Volume testing
  • Stress testing
  • Maintainability testing
  • Compliance testing
  • Efficiency testing
  • Reliability testing
  • Endurance testing
  • Disaster recovery testing
  • Localization testing
  • Internationalization testing

Non-functional testing finds how the performance, reliability, security, and many more, are holding up. these tests are preformed after the Functional tests, and are preformed through automatic testing. It's actually extremely hard to test through manual means.
